WebRTC app6. 
Приложение для звонка с сайта через браузер.

v3.
Пользователи могут заходить одновременно и не пересекаться, связь только с админом.
При вызове пользователем админ получает уведомление, что ему звонят.
На самом деле пользователь только сформировал медиапоток.
По кнопке Админа идет оффер от админа и процедура запускается на сендере.

v4.
добавлены дублирующие файлы, но без видео.
установлено для мастыра

v5.
добавлены socket.id.
промежуточная версия, чтобы сохранить рабочее приложение.
определяется, какой пользователь отключился.
ведется лист ожиданий.

v6.
есть лист ожиданий. 
оффер загружается в память и потом отправляется конкретному пользователю из листа ожиданий.
теперь нужно отправлять кандидатов и отключение также кокретному пользователю,
прекратить кандидатов при соединении,
убирать пользователя из листа ожидания при оффере админа.

v7.
Отключение по кнопке отправляется конкретному пользователю.
Добавлены технические пометки по связи.
Проблема: Если подключено несколько, то из-за потока кандидатов пользователи рассоединяются и вылетают из листа ожидания

v8.
кандидаты отправляются напрямую сендеру и ресейверу. 
Теперь офферы, ансверы и кандидаты не влияют на других пользователей.
Стабильно работает лист ожидания.

server2v2

server2v3

server2v4
